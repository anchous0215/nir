Для детектирования атак был выбран метод, который основан на расстоянии  Махаланобиса. 
Задача состоит в классификации события на текущем временном интервале T:

F(T)→{0,1}#(12)

где 0 — нормальное поведение;
1 — атака.
Для стадии шифрования атаки программы-вымогателя в кластере Kubernetes был разработан следующий метод профилирования поведения системы на основе отслеживания активности файловой системы:
1)Для получения статистически значимых оценок параметров распределения было проведено эталонное профилирование — непрерывный сбор метрик в течение 24 часов, достаточного для фиксации легитимных фоновых процессов кластера. Была сформирована обучающая выборка и собраны векторы наблюдений для периодов штатной работы системы. 
Поскольку интенсивность работы Kubernetes-кластера неравномерна, непрерывный поток данных, на котором будет собираться профиль нормальной работы, был дискретизирован с использованием метода скользящего окна. 
Пусть ∆t — размер временного окна равный 15 минутам. 
Тогда, множество рассматриваемых временных интервалов:

T={T0, T1,T2,...,TN}##(13)
	
где Ti+1=Ti+∆t. 
Рассматриваемые интервалы характеризуется вектором собираемых метрик размерностью 4:

xi=(xi, 1, xi, 2, xi, 3, xi, 4)#(14)

где xi, 1 — CPU(%), среднее использование процессора в процентах, это суммарное время CPU за интервал Δt;

xi, 1=∆container_cpu_usage_seconds_total ∆t∗100%#(15)

-xi, 2 — дисковое пространство (байты), используемые байты на конец интервала; 

xi, 2=container_fs_usage_bytes#(16)

-xi, 3 — I/O (оп/с), количество операций ввода-вывода в секунду;

xi, 3=∆container_fs_io_current ∆t#(17)

-xi, 4 — количество использованных inode.
xi, 4 = container_fs_inodes_total − container_fs_inodes_free#(18)

Сбор наблюдаемых векторов для каждого i-го интервала:
Inode — количество использованных inode. 
2)Вычисление параметров многомерного нормального распределения.
Вектор средних:

μ=(μ1, μ4, μ3, μ4)T#(19)

μj=1Ni=1Nxi, j#(20)

Ковариационная матрица размера 4×4:

Sjk=1N−1i = 1N(xi, j−μj)(xi, k−μk)#(21)

3)Если S вырождена, то есть детерминант близок к нулю, необходимо использовать регуляризацию: 

S=S+λI#(22)

где λ — малая константа (например, 0.01),
I — единичная матрица.
4)Вычисление обратной ковариационной матрицы S−1. 
5)Расчёт расстояния Махаланобиса D для всех временных интервалов:

Di2=(xi−μ)TS−1(xi−μ)#(23)
6)Определение доверительного уровня α в 95%, который является компромиссом между строгостью срабатывания и количеством ложноположительных срабатываний и расчет максимально допустимого расстояния Махаланобиса Dmax2 для отсутствия сигнала об атаке.

Dmax2=max(Di2)∗α#(24)

7)Расчёт расстояния Махаланобиса Dcur2 для временного интервала, в котором проверяется наличие атаки:

Dcur2=(xcur−μ)TS−1(xcur−μ)#(25)

где xcur — вектором собираемых метрик рассматриваемого интервала. 
8)Вычисление результата наличия атаки:

F(T) = Dcur2>Dmax2#(26)

Предложенная методика позволяет обнаруживать попытки несанкционированного доступа к данным, используя логи Kubernetes как источник данных, минимизируя количество ложных срабатываний.
