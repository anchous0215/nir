Для выявления несанкционированного доступа в кластере Kubernetes был разработан метод профилирования поведения систем, который основан на анализе журналов событий контейнеров. Для получения статистически значимых оценок параметров распределения было проведено эталонное профилирование системы, которое состояло в непрерывном сборе метрик в течение одних суток (интервал T0), которого достаточно для фиксации легитимных фоновых процессов кластера, таких как сборка мусора, планировщик задач, ротация логов. Поскольку интенсивность генерации логов в течении суток неравномерна, непрерывный поток данных, на котором было проведено эталонное профилирование, был дискретизирован с использованием метода скользящего окна с размером временного окна равным 1 часу. 
	Поток данных, на котором будет обнаруживаться атака, также дискретизируется с использованием метода скользящего окна с размером временного окна равным 3 минутам., Вляющего оптиманым для обнаружения этапа несанкционированного доступа. 
Пусть ∆t — размер временного окна. 
Тогда, множество рассматриваемых временных интервалов:

T={T0, +T1,T2,...,TM}##(1)

	где Ti+1=Ti+∆t. 
Рассматриваемые интервалы характеризуется множеством лог-событий:

Li={Li, 1,Li, 2,...,Li,  n }##(2)

где n — количество рассматриваемых лог событий в интервале.
Задача состоит в классификации события на i-м рассматриваемом интервале:

F(L)→{0,1}#(3)

где 0 — нормальное поведение;
1 — атака.
Каждое j-е лог-событие в i-м интервале характеризуется набором бинарных признаков:

xi, j=(xi, j, 1, xi, j, 2, ...)#(4)

Тогда множество лог-событий в интервале можно разбить на фактор-множество по отношению эквивалентности равенства набора бинарных признаков:

Li/R=[li, 1/R,li, 2/R, . . .]#(5)

R={(xi, 1, xi, 2)|xi, 1=xi, 2}#(6)

Т.е. лог-события принадлежат одному классу эквивалентности при равенстве всех рассматриваемых полей, кроме времени.
Для того, чтобы вероятность появления лог-событий, которые появляются исключительно во время совершения атаки, не была равна 0 используется сглаживание Лапласа. При появлении в рассматриваемом для обнаружении атаки интервале лог-события, не принадлежащего ни одному классу эквивалентности, в рассматриваемое фактор-множество добавляется класс эквивалентности с данным лог-событием и ко всем остальным классам эквивалентности в фактор-множестве добавляется единожды за рассматриваемый интервал времени один элемент. 
Вычислялась вероятность появления лог-события из k-го класса эквивалентности на i-м интервале при нормальной работе системы:

pi, k=|li, k/R|Δt##(7)
	
Вычисление вероятностей позволяет определить распределение классов вероятностей.
2.3.2 Выбор метода анализа
Критерий χ² хорошо подходит для задачи, потому что он напрямую решает проблему сравнения распределений категориальных данных во времени для дискретного распределения. Критерий χ² предназначен для проверки гипотезы о том, что наблюдаемое распределение не отличается от ожидаемого (базового). Если система работает штатно, структура логов остаётся стабильной, а при атаке происходит перераспределение частот — одни категории начинают резко преобладать, другие исчезают. χ² количественно измеряет это отклонение, суммируя нормированные разницы между наблюдаемыми и ожидаемыми значениями, и позволяет формально определить, является ли изменение статистически значимым, а не случайным шумом. При этом метод не требует предположений о сложной модели данных, легко интерпретируется, масштабируется на большое число категорий и хорошо работает именно с частотными характеристиками, что делает его эффективным выбором для анализа логов.
Критерий χ² в разрабатываемом методе работает следующим образом:
1)Строится гипотеза:
-H₀: атаки нет;
-H₁: возможна атака.
2)Вычисляется число ожидаемых лог-событий из класса эквивалентности k в рассматриваемом интервале, который ставится в соответствие i-му интервалу из эталонного профилирования:
Ei, k=pi, k|L/R|##(8)

3)Рассматриваемое фактор-множествоLi/Rприводится в соответствие обязательному условию применимости критерия χ², который говорит о том, что ожидаемое количество рассматриваемых событий обязано быть не меньше 5. Для этого все Ei, k<5объединяются в один класс эквивалентности, как и соответствующие наблюдаемые лог-события.
4)Вычисляется статистика χ² по следующей формуле:

 χ2=k(Oi, k−Ei, k)2Ei, k##(9)

	где Oi, k — наблюдаемое число элементов из класса эквивалентности k в интервале i;
	Ei, k — ожидаемое число элементов из класса эквивалентности k в интервале i.
5)Определяется количество степеней свободы df:

df =|L/R|−1#(10)

6)Задается уровень значимости α = 0.05, который является компромиссом между строгостью срабатывания и количеством ложноположительных срабатываний.
7)По таблице критических значений распределения χ² определяется критическое значение для данной степени свободы и уровня значимости χi_crit2, которое сравнивается с вычисленным значением χ2. Если вычисленное значение превышает заданное, то гипотеза H₀ отвергается и выдается предупреждение об атаке.
F(L) = χ2>χcrit2#(11)

Предложенная методика позволяет обнаруживать попытки несанкционированного доступа к данным, используя логи Kubernetes как источник данных, минимизируя количество ложных срабатываний.
